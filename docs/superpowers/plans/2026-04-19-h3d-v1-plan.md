`layout.py` content (concrete this step):

```python
from pathlib import Path
from dataclasses import dataclass

EDIT_TYPES_FLUX = ("modification", "scale", "material", "color", "global")
EDIT_TYPES_ALL  = ("deletion", "addition") + EDIT_TYPES_FLUX
N_VIEWS = 5

@dataclass(frozen=True)
class H3DLayout:
    root: Path

    def assets_obj_dir(self, shard, obj_id):    return self.root / "_assets" / shard / obj_id
    def object_npz(self, shard, obj_id):        return self.assets_obj_dir(shard, obj_id) / "object.npz"
    def orig_view(self, shard, obj_id, k):      return self.assets_obj_dir(shard, obj_id) / "orig_views" / f"view{k}.png"
    def edit_dir(self, edit_type, shard, obj_id, edit_id):
                                                 return self.root / edit_type / shard / obj_id / edit_id
    def manifest_path(self, edit_type, shard):  return self.root / "manifests" / edit_type / f"{shard}.jsonl"
    def aggregated_manifest(self):              return self.root / "manifests" / "all.jsonl"
```

**Commit**: `feat(h3d_v1): scaffold cleaning/h3d_v1 package + layout helpers`
**Test**: `tests/cleaning/h3d_v1/test_layout.py` — round-trip path assertions only (~15 lines).

### A2. Implement `filter.py`

Reads pipeline_v3's `edit_status.json` per object, exposes:

```python
def accept_deletion(edit_status: dict, edit_id: str) -> tuple[bool, str | None]: ...
def accept_flux(edit_status: dict, edit_id: str) -> tuple[bool, str | None]: ...
def gate_summary(edit_status: dict, edit_id: str) -> dict:  # for meta.json
    """Return {"gate_A": {...}, "gate_C": ...|None, "gate_E": {...}}"""
```

Filter rules per spec §5:
- deletion: `gate_A.pass == True`; `None`/missing/False ⇒ reject (return reason).
- flux: `gate_A.pass == True AND gate_E.pass == True`; gate_C optional.
- addition: deferred (handled in pull_addition by checking dataset state, not edit_status).

**Commit**: `feat(h3d_v1): gate filter rules with required-null-rejects semantic`
**Test**: `tests/cleaning/h3d_v1/test_filter.py` — table-driven (~30 lines): `[gate_A.pass, gate_E.pass] × {True,False,None,missing}` matrix, assert accept/reject + reason string.

## 2. Stage B — `_assets/` pool

### B1. Implement `asset_pool.py`

Two functions, both fcntl-locked at `_assets/<NN>/<obj_id>/.lock`:

```python
def ensure_object_npz(layout, shard, obj_id, *,
                      pipeline_obj_dir, slat_dir,
                      ss_encoder=None) -> Path:
    """Write _assets/<NN>/<obj_id>/object.npz.

    Sources, in priority order:
      1. <pipeline_obj_dir>/edits_3d/<any flux edit>/before.npz  (already has ss)
      2. slat_dir/<NN>/<obj_id>_*.pt + ss_encoder.encode_ss(coords)  (GPU fallback)

    Returns the materialised path. Idempotent.
    """

def ensure_object_views(layout, shard, obj_id, *, pipeline_obj_dir, image_npz) -> Path:
    """Write _assets/<NN>/<obj_id>/orig_views/view{0..4}.png.

    Sources, in priority order:
      1. <pipeline_obj_dir>/edits_3d/<any add edit>/preview_{0..4}.png  (already correct)
      2. partcraft.render.overview.load_views_from_npz(image_npz, VIEW_INDICES)

    Returns the orig_views directory. Idempotent.
    """
```

GPU fallback for `ensure_object_npz` is wrapped in a lazy import so CPU-only callers never touch CUDA unless that branch fires.

**Commit**: `feat(h3d_v1): asset pool — ensure_object_npz + ensure_object_views with fcntl locking`
**Test**: `tests/cleaning/h3d_v1/test_asset_pool.py` — uses a temp dir + a synthetic mini "pipeline obj dir" (one mock flux edit with a tiny pre-built before.npz; one mock add edit with five 16×16 preview pngs); verifies:
  - both functions are idempotent (second call is no-op);
  - object.npz contains the 3 expected keys;
  - orig_views has 5 files;
  - lock file is created and released.
GPU fallback path is **not** unit-tested (would require encoder + slat_dir); covered by smoke if it fires.

## 3. Stage C — promoter core

### C1. Implement `promoter.py`

Three functions, one per edit-type group. All return a `PromoteResult(ok: bool, reason: str | None, manifest_record: dict | None)`.

```python
def promote_deletion(edit: PipelineEdit, layout: H3DLayout, *, asset_pool_ctx) -> PromoteResult:
    """
    1. asset_pool_ctx.ensure_object_npz(...)    # may copy from flux before.npz
    2. asset_pool_ctx.ensure_object_views(...)  # may copy from add preview
    3. mkdir edit_dir + before_views/ + after_views/
    4. hardlink before.npz, before_views/view{i}.png from _assets
    5. hardlink after.npz from edit_dir/after.npz
    6. hardlink after_views/view{i}.png from edit_dir/preview_{i}.png
    7. write meta.json
    """

def promote_flux(edit, layout, *, asset_pool_ctx) -> PromoteResult: ...

def promote_addition(edit, layout, *, asset_pool_ctx) -> PromoteResult:
    """Requires paired deletion already in dataset.
    Hardlink chain:
      before.npz       <- deletion/.../after.npz
      after.npz        <- _assets/.../object.npz
      before_views/*   <- deletion/.../after_views/*
      after_views/*    <- _assets/.../orig_views/*
    """
```

Internal helper `_hardlink_or_copy(src, dst)`: try `os.link`; on `EXDEV` fall back to `shutil.copy2` with `logger.warning("cross-FS, copied")`. Idempotent (skip if dst exists and inode matches).

**Commit**: `feat(h3d_v1): per-type promoter routines (hardlink + meta.json)`
**Test**: `tests/cleaning/h3d_v1/test_promoter.py` (~80 lines):
  - synthetic pipeline obj dir with 1 del + 1 add + 1 mod, all "passed" gates in a fake `edit_status.json`;
  - run promote_deletion → assert files exist + `os.stat(...).st_ino` equality between before.npz and `_assets/.../object.npz`;
  - run promote_flux on the mod edit → similar inode assertions;
  - run promote_addition → assert add.before.npz inode == del.after.npz inode, add.after.npz inode == object.npz inode;
  - re-run all three → assert no errors (idempotent).

## 4. Stage D — CLI entrypoints

### D1. `scripts/cleaning/h3d_v1/pull_deletion.py`

Thin orchestrator: parse args, load pipeline cfg, build `H3DLayout` + asset_pool ctx, iterate `pipeline_io.iter_edits(types=['deletion'])`, apply `filter.accept_deletion`, call `promote_deletion` per accepted edit, append to manifest. Concurrency via `concurrent.futures.ThreadPoolExecutor` keyed by obj_id (so per-obj asset_pool work doesn't deadlock).

CLI:
```
python -m scripts.cleaning.h3d_v1.pull_deletion \
    --pipeline-cfg <yaml> --shard NN --dataset-root data/H3D_v1 \
    [--workers N] [--dry-run] [--limit N]
```

`--dry-run`: prints accept/reject counts + estimated `_assets` size, writes nothing.

### D2. `scripts/cleaning/h3d_v1/pull_flux.py`

Same shape. Extra arg `--types modification scale material color global` (defaults to all five).

### D3. `scripts/cleaning/h3d_v1/pull_addition.py`

Same shape, simpler (no asset_pool work, just hardlinks). Skip silently when paired deletion missing.

### D4. `scripts/cleaning/h3d_v1/build_h3d_v1_index.py`

```
python -m scripts.cleaning.h3d_v1.build_h3d_v1_index \
    --dataset-root data/H3D_v1 [--validate]
```

Aggregates `manifests/<type>/*.jsonl` → `manifests/all.jsonl`. With `--validate`, walks every record and asserts hardlink target exists + npz is loadable.

### D5. `scripts/cleaning/h3d_v1/pack_shard.py`

```
python -m scripts.cleaning.h3d_v1.pack_shard \
    --dataset-root data/H3D_v1 --shard NN --out releases/H3D_v1__shard<NN>.tar
```

Builds the tarball per spec §6.5 — paths included match spec's pack list exactly. Verify hardlink preservation by inspecting `tar -tvf` output for `h` markers in a final assertion (warn-only, doesn't fail the build).

## 8. Risks & rollback

| Risk | Mitigation | Rollback |
|---|---|---|
| Hardlink target inode mismatch (silent corruption from a later step) | inode assertion in `--validate` + smoke test | `git reset --hard snapshot/pre-h3d-v1-plan-2026-04-19`; `rm -rf data/H3D_v1` |
| GPU fallback in `ensure_object_npz` fires too often (slows promote) | dry-run report (Stage E2) shows count; if >5% of objs need fallback, file an issue and decide whether to pre-cache | none needed — fallback is functionally correct, just slow |
| `_assets/` pool grows in unexpected ways during concurrent runs | fcntl lock + idempotent ensure-* | re-run after fixing; idempotency means no permanent damage |
| Cross-FS write defeats hardlinks | `_hardlink_or_copy` warns on EXDEV; pack_shard final tar still works (just larger) | none — design accommodates this |
| Filter rejects more edits than expected (gate=None semantics) | Stage E2 dry-run surfaces this; can add `--require-gates A` override per CLI if loosening becomes necessary | edit `filter.py` thresholds and re-run |

Per-stage snapshot tags (created on the fly during execution, not now):
- `git tag snapshot/h3d-v1-stage-A-done` after Stage A
- ... etc., used as cheap rollback points if a later stage breaks something earlier.
